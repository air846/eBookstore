# 书城阅读网站开发计划

## 1. 项目目标
基于《开发提示词》落地一个前后端分离的电子书阅读平台，形成可运行的 MVP（后端 + 用户端 + 管理端），并通过分阶段提交保证过程可追溯。

## 2. 阶段划分与交付

### 阶段 1：规划与脚手架初始化
- 明确目录结构与模块边界
- 建立仓库基础文件（.gitignore、说明文档）
- 输出可执行开发计划与里程碑

交付物：
- doc/开发计划.md
- 仓库基础目录：ackend/、library-user/、library-admin/

### 阶段 2：后端核心能力（Spring Boot）
- 初始化 Spring Boot 3 + MyBatis Plus + JWT 工程
- 建立通用响应、统一异常、基础安全鉴权
- 实现核心接口：认证、图书、分类、用户管理、统计
- 提供数据库初始化 SQL（MySQL 8）

交付物：
- 后端可编译源码与配置
- doc/sql/init.sql
- doc/api/ 接口说明

### 阶段 3：用户端（Vue3）
- 初始化用户端项目（Vite + Vue Router + Pinia + Element Plus）
- 完成页面：登录/注册、首页、书籍列表、详情、阅读器、个人中心
- 封装 Axios 与 token 拦截，完成主要接口联调

交付物：
- library-user/ 用户端源码
- 页面路由与状态管理

### 阶段 4：管理端（Vue3）
- 初始化管理端项目（Vite + Vue Router + Pinia + Element Plus）
- 完成页面：登录、仪表盘、书籍管理、分类管理、用户管理、系统设置
- ECharts 接入关键统计图

交付物：
- library-admin/ 管理端源码
- 管理侧接口联调与基础权限控制

## 3. 里程碑提交策略
- eat: phase1 项目规划与基础结构初始化
- eat: phase2 后端核心模块与数据库脚本
- eat: phase3 用户端核心页面与接口对接
- eat: phase4 管理端核心页面与联调文档

## 4. 验收标准（MVP）
- 用户端可完成注册登录、浏览书籍、阅读与收藏
- 管理端可完成登录、图书/分类/用户基本管理与统计查看
- 后端提供统一 API、JWT 鉴权、基础权限校验
- 数据库可通过初始化脚本一键建表与导入示例数据
